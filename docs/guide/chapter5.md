# 第5章：实战项目

## 5.1 项目1：29秒拆分工资条

> **来源**：金田的真实案例
> **难度**：⭐
> **用时**：29秒
> **效果**：自动完成重复性工作

---

### 场景描述

作为老板和财务，每月需要给团队发工资条。

**问题**：
- 代理只给我一个 Excel 工资表（所有人在一起）
- 需要拆成独立的工资条（每个人一个文件）
- 然后分别发送给每个人

**以前**：手动复制粘贴，每次30分钟，很烦。

**现在**：29秒搞定！

---

### 实施步骤

#### 步骤1：准备工资表

创建一个简单的 Excel 文件 `salary.xlsx`：

| 姓名 | 部门 | 基本工资 | 奖金 | 扣款 | 实发工资 |
|------|------|---------|------|------|----------|
| 张三 | 技术 | 10000 | 2000 | 500 | 11500 |
| 李四 | 销售 | 8000 | 3000 | 200 | 10800 |
| 王五 | 运营 | 9000 | 1500 | 300 | 10200 |

#### 步骤2：启动 Claude Code

```bash
cd 金田工作室/工资条处理
claude
```

#### 步骤3：拖拽文件并提问

**把 `salary.xlsx` 拖到 Claude Code 里**，然后输入：

```
这是一个工资表，帮我拆成工资条，一个人一个文件
```

#### 步骤4：等待结果（29秒）

Claude Code 会：
1. 读取 Excel 文件
2. 创建 Python 脚本处理数据
3. 生成独立的工资条文件（`张三工资条.md`、`李四工资条.md`...）
4. 完成！

---

### 完整对话示例

```
你：这是一个工资表，帮我拆成工资条，一个人一个文件

Claude Code：好的，我来帮你处理。

[读取 salary.xlsx]

我看到这个工资表包含：姓名、部门、基本工资、奖金、扣款、实发工资

我将：
1. 读取整个表格
2. 为每个人创建一个独立的工资条文件
3. 使用 Markdown 格式，方便查看

[创建脚本并执行]

✅ 已完成！生成了 3 个工资条文件：
- 张三工资条.md
- 李四工资条.md
- 王五工资条.md

每个文件包含该员工的完整工资信息。
```

---


---

### 关键技巧

1. **拖拽文件**：直接把文件拖到 Claude Code，它会自动读取
2. **明确需求**："一个人一个文件"比"处理这个表"更清晰
3. **信任 AI**：不要干预，让它自己完成

## 5.2 项目2：自动生成周报

> **难度**：⭐⭐
> **用时**：2分钟
> **效果**：每周节省30分钟

---

### 场景描述

每周需要写工作周报，内容包括：
- 本周完成的工作
- 遇到的问题
- 下周计划

**问题**：每次都要回忆、整理、格式化，很费时间。

**解决方案**：让 Claude Code 自动从项目文件中提取信息！

---

### 实施步骤

#### 步骤1：准备项目上下文

在你的项目文件夹创建 `CLAUDE.md`：

```markdown
# 项目周报生成规则

## 项目信息
- 项目名称：AI 视频生成工具
- 我的角色：全栈开发
- 团队规模：3人

## 周报格式
### 本周完成
- [功能1]
- [功能2]

### 遇到的问题
- [问题1] → [解决方案]

### 下周计划
- [计划1]
- [计划2]
```

#### 步骤2：让 Claude Code 生成周报

```bash
cd 金田工作室/ai-video-tools
claude
```

**输入**：
```
请基于我的 Git 提交记录和项目文件，生成本周工作周报
```

#### 步骤3：检查并调整

Claude Code 会：
1. 读取 `git log`（本周的提交）
2. 分析改动的文件
3. 总结完成的功能
4. 生成格式化的周报

**示例输出**：

```markdown
# 工作周报 - 2026年第6周

## 本周完成

### 功能开发
- ✅ 完成视频生成 API 集成
- ✅ 实现字幕自动生成功能
- ✅ 添加进度条显示

### Bug 修复
- 🐛 修复视频导出卡死的问题
- 🐛 修复字幕时间轴错误

### 代码质量
- 📝 添加单元测试覆盖核心功能
- 🧹 重构 API 调用逻辑

---

## 遇到的问题

### 问题1：视频生成超时
**现象**：大视频生成时经常超时
**解决**：实现异步任务队列

### 问题2：字幕编码错误
**现象**：中文字幕乱码
**解决**：统一使用 UTF-8 编码

---

## 下周计划

- [ ] 实现视频模板功能
- [ ] 优化生成速度（目标：提升50%）
- [ ] 添加用户反馈收集
- [ ] 编写 API 文档
```

---

### 关键技巧

1. **利用 Git 历史**：`git log` 是最好的工作记录
2. **提前定义格式**：在 `CLAUDE.md` 中写清楚周报格式
3. **让 AI 主动挖掘**：不只是汇总，还要分析

## 5.4 项目4：自动回复邮件

> **难度**：⭐⭐⭐
> **用时**：5分钟（设置一次，永久使用）
> **效果**：每天节省1小时

---

### 场景描述

作为小老板，每天收到很多邮件：
- 客户咨询
- 合作邀请
- 账单通知
- 垃圾邮件

**问题**：回复邮件占用大量时间。

**解决方案**：训练 Claude Code 自动回复！

---

### 实施步骤

#### 步骤1：创建回复规则

**文件：`email-rules.md`**

```markdown
# 邮件自动回复规则

## 客户咨询
- 回复时间：2小时内
- 语气：专业、友好
- 内容：解答问题 + 引导购买

## 合作邀请
- 回复时间：24小时内
- 语气：开放、谨慎
- 内容：了解详情 + 约会议

## 账单通知
- 回复时间：即时
- 语气：简洁
- 内容：确认收到

## 垃圾邮件
- 回复时间：不回复
- 操作：标记为垃圾邮件
```

#### 步骤2：创建回复脚本

**让 Claude Code 生成**：

```
请基于这些规则，创建一个邮件自动回复脚本
要求：
1. 读取邮箱新邮件
2. 分类邮件类型
3. 生成草稿回复
4. 我来确认后发送
```

#### 步骤3：使用

每天早上打开 Claude Code：

```
请检查今天的新邮件并生成回复草稿
```

**然后你只需要**：
1. 快速浏览草稿
2. 点确认发送
3. 1小时工作 → 5分钟搞定！

## 5.5 项目5：数据分析助手

> **难度**：⭐⭐⭐⭐
> **用时**：30分钟（首次设置）
> **效果**：自动生成数据报告

---

### 场景描述

你的业务需要分析：
- 销售数据
- 用户行为
- 广告效果

**问题**：不会 Python/R，Excel 也用得不熟。

**解决方案**：Claude Code = 你的数据分析师！

---

### 实施步骤

#### 步骤1：准备数据

**文件：`sales.csv`**

```csv
日期,产品,销量,收入
2026-01-01,产品A,100,10000
2026-01-01,产品B,50,5000
2026-01-02,产品A,120,12000
...
```

#### 步骤2：让 Claude Code 分析

```
请分析这个销售数据：
1. 总销售额和趋势
2. 最畅销的产品
3. 生成可视化图表
```

#### 步骤3：查看结果

Claude Code 会：
1. 读取 CSV 文件
2. 计算：总销售额、平均值、增长率
3. 生成 Python 脚本画图
4. 输出图表文件

**示例输出**：

```markdown
# 销售数据分析报告

## 总体情况

- 总销售额：¥150,000
- 平均日销：¥5,000
- 增长趋势：+15% 📈

## 产品表现

| 产品 | 销量 | 收入 | 占比 |
|------|------|------|------|
| 产品A | 500 | 50,000 | 33% |
| 产品B | 300 | 30,000 | 20% |
| 产品C | 700 | 70,000 | 47% |

**最畅销**：产品C（47%市场份额）

## 趋势分析

[生成折线图：sales-trend.png]
- 整体上升趋势
- 周末销量较高

## 建议

1. 加大产品C的推广（表现最好）
2. 产品B需要优化（销量低）
3. 周末增加营销活动
```

---

### 关键技巧

1. **明确分析目标**：不要只说"分析数据"，要说"分析什么"
2. **要求可视化**：让 AI 生成图表，不只是数字
3. **要建议**：不只是分析，还要给出行动建议

## 5.6 综合项目：AI 博主工作流

> **难度**：⭐⭐⭐⭐⭐
> **用时**：1小时（首次设置）
> **效果**：自动化内容生产

---

### 场景描述

你想做一个 AI 相关的博主：
- 每周发布 3 篇文章
- 每周发布 2 个视频
- 每天发布 5 条社交媒体

**问题**：一个人忙不过来！

**解决方案**：Claude Code + 多个 AI 工具 = 自动化内容工厂

---

### 工作流设计

```
灵感收集 → 内容创作 → 多平台分发 → 数据分析
    ↓         ↓            ↓            ↓
  Kimi    Claude Code   自动化脚本    Google Sheets
```

---

### 实施步骤

#### 阶段1：灵感收集

**让 Kimi 每天搜索热点**：

```bash
# 每天早上自动运行
npx @zai/collect-trends --topic="AI工具" --output=ideas.md
```

**输出**：
```markdown
## 今日热点（2026-02-07）

1. OpenAI 发布新模型 GPT-5
2. Claude Code 更新 Skills 功能
3. Kimi 开放视频生成 API
...
```

---

#### 阶段2：内容创作

**让 Claude Code 写文章**：

```
基于 ideas.md 的第2个热点，写一篇1000字的教程文章
要求：
- 通俗易懂
- 有代码示例
- 适合发布在知乎
```

**输出**：`articles/2026-02-07-claude-code-skills.md`

---

#### 阶段3：多平台分发

**自动适配格式**：

```
把这篇文章转换成：
1. 知乎文章格式（Markdown）
2. 微信公众号格式（带排版）
3. 视频脚本格式（分镜）
4. 社交媒体短文（微博/Twitter）
```

**一键发布到多个平台**！

---

#### 阶段4：数据分析

**每周自动生成报告**：

```
分析本周所有文章/视频的数据表现
- 阅读量/播放量
- 互动率（点赞、评论、转发）
- 用户反馈

总结哪些内容更受欢迎，下周优化方向
```
